Practical Guide – BENYS

How to use the application step by step

Understanding the Dashboard

  • On the left → your projects
  • On the right → your assigned tasks

Click on:

  • a project → to see its details
  • a task → to view/edit its information

Access your projects

Click on Projects (in the top blue bar)

You will see all your projects

Click on a project to open it or click the OPEN button

Permissions

A standard user can only:

  • View projects they are a member of
  • View tasks in those projects

An administrator must grant you additional permissions to access more features.

Admin Guide

Create a project

  • Click + New Project
  • Fill in the information
  • Validate

Manage members

  • Open a project
  • Go to Team Members and click either Manage or Add some
  • Add a user
  • Choose a role:
    • Contributor
    • Read-only
      You can always change the role of a member any time

Remove a user

Result:

  • The user loses access
  • Their tickets remain but become unassigned (the user will no longer see them)

Create and manage a Sprint

Create

  • Open a project
  • Click + New Sprint

    • You will need to add:
    Name + description (optional) + start date + duration (14 days by default, but you can change it)
    The end date is calculated automatically

Start

The sprint becomes active. In the left sidebar, you can click to view the active sprint in the KANBAN

Close

The sprint is permanently closed. Unfinished tasks will be transferred to the next sprint

Create a Ticket

You chose the type of the ticket and add the additionnal informations

Product Backlog

  • View unplanned tasks
  • Organize priorities

Kanban (Sprint)

Drag and drop tickets

Epic

Create an epic :
you can add color + dates
Group multiple tickets

Quick navigation (left sidebar)

  • Overview → summary
  • Backlog → tasks
  • Active Sprint → kanban
  • Issues → tickets
  • Epic → epics
